

## Скрытие клиента из дашборда

### Что будет сделано
Добавить возможность скрыть клиента из списка и статистики на дашборде админа. Скрытый клиент остаётся в системе, но не учитывается в статистике и не отображается в основном списке.

### Как это будет работать
- На странице клиента (AdminClientView) появится кнопка "Скрыть из дашборда" / "Показать в дашборде" рядом с кнопками блокировки/удаления
- На дашборде (AdminDashboard) скрытые клиенты не отображаются в списке и не учитываются в статистике
- Опционально: внизу списка клиентов будет сворачиваемый блок "Скрытые" для быстрого доступа к скрытым клиентам

### Технические детали

**1. Миграция базы данных**
Добавить колонку `is_hidden` в таблицу `profiles`:
```sql
ALTER TABLE public.profiles ADD COLUMN is_hidden boolean NOT NULL DEFAULT false;
```

**2. Файл: `src/pages/admin/AdminDashboard.tsx`**
- Добавить в интерфейс `ClientSummary` поле `is_hidden: boolean`
- Разделить клиентов на видимых и скрытых: `visibleClients` и `hiddenClients`
- Статистику считать только по видимым клиентам
- Добавить сворачиваемый блок "Скрытые" внизу списка (если есть скрытые)

**3. Файл: `src/pages/admin/AdminClientView.tsx`**
- Добавить кнопку-переключатель "Скрыть / Показать" в шапке страницы клиента
- При нажатии обновлять `is_hidden` в таблице `profiles` через Supabase

Никаких новых зависимостей не требуется.
